현재 변경사항을 분석하고 커밋을 생성해주세요:

## 절차

1. `git status`로 변경된 파일 확인 (untracked 포함)
2. `git diff`와 `git diff --cached`로 변경 내용 파악
3. `git log --oneline -5`로 최근 커밋 스타일 확인
4. 변경사항의 성격에 맞는 타입을 선택하고 커밋 메시지 작성
5. 관련 파일만 `git add`로 스테이징 (`git add .` 사용 금지)
6. 커밋 생성

## 커밋 메시지 규칙

- 형식: `<타입>: <제목>`
- 제목은 50자 이내, 한국어로 작성
- 무엇을/왜 했는지 중심으로 작성

### 타입

| 타입     | 설명                                |
| -------- | ----------------------------------- |
| feat     | 새로운 기능                         |
| fix      | 버그 수정                           |
| docs     | 문서 변경                           |
| style    | 코드 포맷팅 (세미콜론, 들여쓰기 등) |
| refactor | 코드 리팩토링                       |
| perf     | 성능 개선                           |
| test     | 테스트 추가/수정                    |
| chore    | 빌드, 설정 변경                     |

## 원자적 커밋 (Atomic Commit)

- 하나의 커밋은 하나의 논리적 변경만 포함해야 한다
- 변경사항이 여러 성격이면 타입별로 분리하여 각각 커밋한다
- 커밋 단위 기준: 기능 추가, 버그 수정, 리팩토링, 설정 변경 등은 별도 커밋으로 분리
- 각 커밋은 단독으로 빌드/실행 가능한 상태를 유지해야 한다
- 큰 작업은 의미 있는 중간 단계로 쪼개어 커밋한다

## 주의사항

- `.env`, 시크릿 파일은 절대 커밋하지 않음
- 본문이 필요한 경우 제목과 빈 줄로 구분하고 72자마다 줄바꿈
