현재 프로젝트의 성능을 분석하고 최적화 방안을 제안해주세요:

## 분석 대상

### 1. 번들 및 로딩

- `next/dynamic`을 활용한 지연 로딩이 필요한 클라이언트 컴포넌트 식별
- `"use client"` 지시어가 불필요하게 사용된 컴포넌트 확인
- 서드파티 라이브러리 중 tree-shaking이 안 되는 import 패턴 검출

### 2. 렌더링 전략

- Server Component로 전환 가능한 Client Component 식별
- 불필요한 리렌더링을 유발하는 패턴 확인 (인라인 객체/함수, Context 범위 등)
- `useCallback`/`useMemo`가 필요하지만 누락된 Context Provider value 확인

### 3. 이미지 및 폰트

- `next/image` 미사용 `<img>` 태그 검출
- 폰트 로딩 최적화 상태 확인 (`display: swap`, preload 등)

### 4. CSS

- Tailwind CSS에서 불필요한 임의값(`[...]`) 사용 여부
- 중복되거나 상충하는 클래스 검출

### 5. 데이터 패칭

- fetch 캐싱 전략 확인 (revalidate, cache 옵션)
- 워터폴 요청 패턴 검출 (순차 fetch를 병렬로 전환 가능한 경우)

## 출력 형식

각 항목에 대해 다음 형식으로 보고:

- **파일**: 해당 파일 경로
- **문제**: 현재 상태와 성능 영향
- **제안**: 구체적인 개선 방법 (코드 예시 포함)
- **우선순위**: 높음 / 보통 / 낮음

우선순위가 높은 항목부터 정렬하여 보고해주세요.
